# Transcribator

MVP веб-приложения для транскрибации видео и аудио с таймкодами и структурированным саммари для рабочих встреч.

## Что умеет

- принимает аудио и видеофайлы через браузер;
- поддерживает drag-and-drop без открытия файла в соседней вкладке;
- автоматически извлекает аудио из видео;
- сжимает аудио перед отправкой в OpenAI;
- делит большой подготовленный аудиофайл на чанки, если он все еще слишком большой;
- строит полный транскрипт с таймкодами;
- формирует обзор встречи и TL;DR;
- выделяет цель созвона, решения, договоренности, открытые вопросы и следующие шаги;
- показывает список action items с ответственными и сроками, если они найдены в разговоре.

## Технологии

- Node.js
- Express
- Multer
- OpenAI Node SDK
- ffmpeg-static
- ffprobe-static

## Быстрый старт

1. Установи зависимости:

```powershell
npm.cmd install
```

2. Создай `.env` на основе примера:

```powershell
Copy-Item .env.example .env
```

3. Укажи ключи:

```env
OPENAI_API_KEY=your_openai_api_key_here
DEEPGRAM_API_KEY=your_deepgram_api_key_here
```

4. Запусти приложение:

```powershell
npm.cmd start
```

5. Открой [http://localhost:3000](http://localhost:3000)

Или просто запусти [start-app.bat](c:\Users\Askay\Desktop\AI\transcribator\start-app.bat) двойным кликом: он откроет сервер и браузер автоматически.

## Переменные окружения

- `OPENAI_API_KEY` - обязательный ключ OpenAI API для summary
- `DEEPGRAM_API_KEY` - обязательный ключ Deepgram API для транскрибации
- `PORT` - порт сервера, по умолчанию `3000`
- `DEEPGRAM_MODEL` - по умолчанию `nova-3`
- `DEEPGRAM_LANGUAGE` - по умолчанию `ru`
- `SUMMARY_MODEL` - по умолчанию `gpt-4o-mini`
- `MAX_UPLOAD_MB` - лимит файла, который принимает сам сервер, по умолчанию `512`
- `AUDIO_BITRATE` - битрейт подготовленного аудио, по умолчанию `48k`
- `AUDIO_SAMPLE_RATE` - sample rate подготовленного аудио, по умолчанию `16000`

## Ограничения MVP

- Deepgram получает уже подготовленное и при необходимости разрезанное аудио, поэтому сервер сохраняет preprocessing через `ffmpeg` и чанки.
- Для таймкодов используются `utterances`, а при их отсутствии сервер собирает сегменты из слов Deepgram.
- Структурированное саммари строится поверх уже склеенного транскрипта и зависит от качества исходной записи и ясности разговора.
